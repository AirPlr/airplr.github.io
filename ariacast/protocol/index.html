📡 AriaCast Protocol

AriaCast is a lightweight, bidirectional audio streaming protocol designed for real-time, low-latency audio transmission over local networks. Built as a modern alternative to AirPlay and VBAN, AriaCast relies on UDP for rapid discovery and TCP (WebSockets/HTTP) for reliable streaming and control.

🏗 Network Architecture

AriaCast operates on a standard Client-Server model. The Client (e.g., your smartphone) captures local audio and streams it to the Server (the receiver), which processes the incoming PCM audio and handles playback control.

Service Port (Default) Protocol Direction
Discovery 12888 UDP Broadcast
Streaming & API 12889 TCP (WS/HTTP) Bidirectional

🔄 Protocol Flow

1. Discovery (UDP)

To find available receivers on the network, the client broadcasts a discovery packet.

Client Broadcast:

DISCOVER_AUDIOCAST

Server Response:

The server replies with a JSON payload detailing its connection parameters:

{
  "server_name": "AriaCast Speaker",
  "ip": "192.168.1.100",
  "port": 12889,
  "samplerate": 48000,
  "channels": 2
}

2. Connection & Handshake (WebSocket)

Once discovered, the client establishes a WebSocket connection to the /audio endpoint. The server immediately responds with a handshake configuration to align audio settings.

Server → Client Handshake:

{
  "status": "READY",
  "type": "handshake",
  "sampleRate": 48000,
  "channels": 2,
  "sampleWidth": 2,
  "frameSize": 3840
}

3. Audio Streaming

Audio is transmitted as raw binary data through the established WebSocket.

💡 Audio Format Details:

  • Encoding: PCM 16-bit Signed Little-Endian, Stereo, 48kHz.
  • Packet Size: Exactly 3840 bytes per frame (representing 20ms of audio).
  • Buffering: AriaCast typically uses a 300ms pre-buffer to handle local network jitter effectively.

🔌 API Endpoints

AriaCast uses a mix of standard HTTP POST requests for state and WebSockets for real-time control.

Sync Metadata

POST /metadata
Pushes real-time track information from the client to the server.

Payload:

{
  "data": {
    "title": "Song Title",
    "artist": "Artist Name",
    "album": "Album Name",
    "artworkUrl": "https://example.com/cover.jpg",
    "durationMs": 180000,
    "positionMs": 45000,
    "isPlaying": true
  }
}

Remote Control

WS /control
Enables bidirectional remote control. The server/receiver UI can send commands back to the source app.

Supported Messages (Server → Client):
play, pause, next, previous, toggle, stop

Health & Stats

WS /stats
The server pushes playback health statistics back to the client for debugging and quality monitoring.

Payload:

{
  "bufferedFrames": 15,
  "droppedFrames": 0,
  "receivedFrames": 12345
}
